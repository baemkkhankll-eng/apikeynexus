ระบบตรวจสอบสิทธิ์คีย์ตรวจสอบฮาร์ดแวร์ไอดี (HWID)

กลับไปแดชบอร์ด

ภาพรวมระบบ (Overview)

API ของเราถูกออกแบบมาบนสถาปัตยกรรมแบบ RESTful ที่มีน้ำหนักเบาและมีความปลอดภัยสูง เพื่อช่วยให้คุณสามารถตรวจสอบคีย์ลิขสิทธิ์ซอฟต์แวร์ที่ล็อกติดกับ Hardware ID (HWID) ประจำเครื่องของลูกค้ารองรับการทำงานบนเครื่องเดียวต่อหนึ่งคีย์อย่างแม่นยำ ป้องกันการนำคีย์ไปแชร์ใช้งานต่อได้อย่างเด็ดขาด

Base URL https://apikeynexus.onrender.com/

การอ้างอิงและคู่มือสำหรับภาษา Python

1. ติดตั้งไลบรารีที่จำเป็น

pip install requests

2. ตัวอย่างการเขียนสคริปต์ตรวจสอบแบบเต็ม (Full Implementation)

import uuid
import hashlib
import requests

class HWIDAuth:
    def __init__(self, api_url, developer_id):
        self.api_url = api_url
        self.developer_id = developer_id
    
    def get_hwid(self):
        # ดึงค่า MAC Address ของเครื่องมาแฮชเพื่อสร้างคีย์เฉพาะตัวของฮาร์ดแวร์
        mac = uuid.getnode()
        return hashlib.sha256(str(mac).encode()).hexdigest()[:32]
    
    def validate(self, key):
        hwid = self.get_hwid()
        try:
            response = requests.post(
                f"{self.api_url}/api/validate",
                json={
                    "key": key,
                    "hwid": hwid,
                    "developerId": self.developer_id
                },
                timeout=10
            )
            return response.json()
        except Exception as e:
            return {"valid": False, "message": str(e)}

# --- ตัวอย่างการเรียกใช้งานจริง ---
API_URL = "https://apikeynexus.onrender.com"
DEVELOPER_ID = "YOUR_DEVELOPER_ID" # คัดลอกค่าจากหน้าแดชบอร์ดของคุณ
LICENSE_KEY = "KEY-XXXX-XXXX-XXXX-XXXX"

auth = HWIDAuth(API_URL, DEVELOPER_ID)
result = auth.validate(LICENSE_KEY)

if result["valid"]:
    print("✓ ลิขสิทธิ์ถูกต้อง! กำลังเปิดโปรแกรม...")
    # วางฟังก์ชันเริ่มทำงานระบบของคุณตรงนี้
else:
    print(f"✗ เข้าสู่ระบบล้มเหลว: {result['message']}")
    exit(1)

การอ้างอิงและคู่มือสำหรับภาษา Node.js / JavaScript

1. ติดตั้งแพ็คเกจผ่าน npm

npm install node-machine-id axios

2. ตัวอย่างการเขียนสคริปต์แบบเต็มสำหรับ Backend / Desktop App

const { machineIdSync } = require('node-machine-id');
const crypto = require('crypto');
const axios = require('axios');

class HWIDAuth {
  constructor(apiUrl, developerId) {
    this.apiUrl = apiUrl;
    this.developerId = developerId;
  }
  
  getHWID() {
    // เจนคีย์ประจำเมนบอร์ดหรือระบบปฏิบัติการแบบถาวร
    const id = machineIdSync();
    return crypto.createHash('sha256').update(id).digest('hex').substring(0, 32);
  }
  
  async validate(key) {
    const hwid = this.getHWID();
    try {
      const response = await axios.post(`${this.apiUrl}/api/validate`, {
        key: key,
        hwid: hwid,
        developerId: this.developerId
      });
      return response.data;
    } catch (error) {
      return { valid: false, message: error.message };
    }
  }
}

// --- วิธีการสั่งรันตรวจสอบ ---
const auth = new HWIDAuth("https://apikeynexus.onrender.com", "YOUR_DEVELOPER_ID");

async function startApp() {
  const result = await auth.validate("KEY-XXXX-XXXX-XXXX-XXXX");
  
  if (result.valid) {
    console.log("✓ License valid! Application starting...");
  } else {
    console.log(`✗ License invalid: ${result.message}`);
    process.exit(1);
  }
}

startApp();

เอกสารอ้างอิง REST API Endpoint

POST /api/validate
ตรวจสอบและล็อกข้อมูลฮาร์ดแวร์ไอดี

⚡ Request Payload (JSON)

{
  "key": "KEY-XXXX-XXXX-XXXX-XXXX",
  "hwid": "32-character-hwid-hash",
  "developerId": "DEV-XXXXXXXX"
}

🟢 Response (เมื่อคีย์ผ่านการยืนยัน)

{
  "valid": true,
  "message": "Key verified successfully"
}

🔴 Response (กรณีคีย์ไม่ผ่าน/ถูกระงับ)

{
  "valid": false,
  "message": "Key is banned or expired"
}

คำอธิบายข้อความปฏิเสธสิทธิ์ (Error Responses)

ค่ากล่องข้อความที่ส่งกลับ สาเหตุ / ความหมาย
Invalid key ไม่มีชุดเลขคีย์นี้อยู่ในฐานข้อมูลระบบ
Key is banned คีย์ถูกสั่งระงับสิทธิ์ชั่วคราวหรือถาวรโดยผู้พัฒนา
Key expired ระยะเวลาโควตาการใช้งานของคีย์ตัวนี้หมดลงแล้ว
HWID mismatch คีย์นี้ถูกล็อกกับอุปกรณ์เครื่องอื่นไปแล้ว ห้ามเปลี่ยนเครื่อง
Developer ID mismatch Developer ID ผู้เรียกใช้ระบบกับตัวรับปลายทางไม่ตรงกัน

ข้อกำหนดและข้อควรระวังสำคัญ (Security Protocols)

  • ▪ ห้ามเปิดเผยค่า Developer ID แก่บุคคลภายนอกโดยเด็ดขาด เนื่องจากเป็นคีย์หลักที่ระบุสิทธิ์ความเป็นเจ้าของระบบจัดการคีย์ทั้งหมด
  • ▪ ฟังก์ชันการคำนวณค่า HWID ภายในโปรแกรม (Client-side) จะต้องคงที่และไม่ผันแปรตามตำแหน่งที่อยู่ไฟล์ เพื่อให้ระบบจำเครื่องเดิมได้อย่างสมบูรณ์
  • ▪ เมื่อมีการใช้งานคีย์ครั้งแรก ระบบหลังบ้านจะบันทึกรหัสฮาร์ดแวร์ไอดีเครื่องนั้นทันที (First-use Hardware Lock)
  • ▪ หากลูกค้าของท่านทำการเปลี่ยนอุปกรณ์คอมพิวเตอร์ หรืออัปเกรดชิ้นส่วนหลัก ท่านสามารถดำเนินการกดปุ่ม "รีเซ็ต HWID" ผ่านทางหน้าแดชบอร์ดเพื่อให้คีย์กลับมาว่างพร้อมลงทะเบียนเครื่องใหม่อีกครั้ง
